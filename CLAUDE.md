# WomenAId

## О проекте

WomenAId — это AI-триаж снимков шейки матки, работающий как decision-support
для медперсонала (НЕ как постановка диагноза), вместе с модулем мониторинга
скрининга и симптомов для пациенток в Казахстане. Проект на стадии MVP и
готовится к пилоту с клиникой.

## Неприкосновенные правила

Эти правила касаются безопасности пациенток и обязательны к соблюдению. Любое
их изменение — явный стоп-сигнал: не вносить молча, сначала спросить пользователя
(см. последний пункт).

- **AI-модуль никогда не возвращает «диагноз»** — только триаж-категорию:
  `INSUFFICIENT_QUALITY` / `ROUTINE_FOLLOWUP` / `PRIORITY_REVIEW` / `URGENT_REVIEW`.

- **Эндпоинт для пациентки (`/risk-assessment/upload`) НИКОГДА не возвращает
  `raw_score` / `confidence`** — только `triage_label` + `patient_facing_message`
  + `disclaimer`. Полный score доступен исключительно в
  `/risk-assessment/clinic/queue`.

- **Ни один `patient_facing_message` не может означать «вы здоровы, к врачу
  не нужно».** Допустимые формулировки заданы в
  `app/ml/inference.py::PATIENT_MESSAGES` — придерживаться их.

- **Red-flag симптомы (`app/models/screening_rules.py::RED_FLAG_SYMPTOMS`)
  всегда дают рекомендацию «к врачу»** — независимо от вывода AI и независимо
  от расписания скрининга.

- **Любая запись о risk assessment должна нести поле статуса модели**
  (сейчас `model_status`), явно показывающее, что модель не прошла клиническую
  валидацию, — пока это так.

- **Без активного согласия пациентки — AI-анализ невозможен технически, не
  только по политике.** Эндпоинт `/risk-assessment/upload/{patient_id}` обязан
  жёстко отклонять запрос, если у пациентки нет активного согласия
  (`consent_records` без `withdrawn_at`); это гейт на уровне кода
  (`app/consent.py::require_active_consent`), а не только организационное правило.

- **Любое изменение этих правил — явный стоп-сигнал.** Не делать молча,
  спросить пользователя.

## Архитектура

```
backend/
  app/
    db/        — слой доступа к данным (SQLite)
    models/    — ORM-модели, screening_rules.py (RED_FLAG_SYMPTOMS)
    schemas/   — Pydantic-схемы запросов/ответов
    routers/   — FastAPI-роутеры (/risk-assessment/upload, /clinic/queue, ...)
    ml/        — инференс и обучение, inference.py (PATIENT_MESSAGES), train.py
frontend/
  index.html   — статический фронтенд без сборки
```

- **Хранилище:** SQLite.
- **ML:** scikit-learn — `RandomForest` на классических image-фичах, **не CNN**.
  Это сознательное решение под ограничения окружения; путь апгрейда (переход
  на CNN при наличии реальных данных и ресурсов) описан в `README.md`.

## Текущий статус данных

Модель обучена на **синтетических** данных
(`dataset_status=DEMO_SYNTHETIC_NOT_CLINICAL`). Реальных клинических данных
пока **нет** — идут переговоры с клиниками. До получения и валидации на
реальных данных любой вывод модели носит демонстрационный характер
(см. правило про `model_status`).

## Команды

Поднять backend (из каталога `backend/`):

```bash
uvicorn app.main:app --reload
```

Сгенерировать демо-данные и обучить модель:

```bash
python -m app.ml.make_demo_data   # сгенерировать синтетический датасет
python -m app.ml.train            # обучить RandomForest на сгенерированных данных
```

Открыть фронтенд — статический `frontend/index.html` без сборки, например:

```bash
python -m http.server --directory frontend 8080
# затем открыть http://localhost:8080 в браузере
```
